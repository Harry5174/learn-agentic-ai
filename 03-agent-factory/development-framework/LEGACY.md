The Agent Factory Development Framework (AFDF) in this directory was the valid framework location during earlier Agent Factory work. It is now frozen as historical material; the [Agent Factory Engineering Framework (AFEF)](https://github.com/Harry5174/agent-factory-engineering-framework) is the canonical successor.

## Canonical Successor

- Canonical repository: [Agent Factory Engineering Framework](https://github.com/Harry5174/agent-factory-engineering-framework)
- First canonical public baseline: [AFEF v0.1.0 release](https://github.com/Harry5174/agent-factory-engineering-framework/releases/tag/v0.1.0)
- Exact v0.1.0 commit: [`79eba2a146d9be8a8502d2d91e067e8d795e8945`](https://github.com/Harry5174/agent-factory-engineering-framework/commit/79eba2a146d9be8a8502d2d91e067e8d795e8945)
- Current released version: consult the [AFEF Releases page](https://github.com/Harry5174/agent-factory-engineering-framework/releases)

AFEF v0.1.0 established the first canonical public baseline. A version declaration or this historical pointer does not determine the current released version; verify the applicable AFEF release and Git refs.

## Historical Purpose

This directory is retained because it contains:

- The original AFDF specifications and templates.
- Historical framework evidence.
- Historical Agent Factory project memory.
- Artifact bootstrap examples.
- The source material from which AFEF was selectively extracted.

These records preserve project history and provenance. Freezing them does not make their earlier use incorrect.

## Freeze Rule

- Do not implement new framework capabilities here.
- Do not synchronize future AFEF releases into this directory.
- Do not use its bootstrap files for new projects.
- Do not treat its status documents as current AFEF status.
- Do not delete it; it preserves project history and provenance.

Historical files may be read when reproducing or studying earlier Agent Factory work. Current framework work begins from the canonical AFEF repository.

## Project-Memory Boundary

- Historical `project-memory/` records belong to `learn-agentic-ai`.
- They were not migrated into canonical AFEF and are not framework-owned operational state.
- Future adopting projects create, pin, and own their own operational adoption records.
- The GitHub Steward repository will eventually own its own pinned AFEF adoption records.
- GitHub Steward adoption has not started.

Do not copy these historical records into AFEF or treat them as current GitHub Steward memory.
